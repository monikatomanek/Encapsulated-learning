Dotenv Guide - Protecting Sensitive Info

This guide is part of the Encapsulated-learning repo by Monika Tomanek. Do not redistribute without attribution.

Why do we need this?

When you write code for apps, APIs, or databases, you often need to use private information â€” things like:

  • Database usernames and passwords
  • API keys
  • Server addresses

If you include these directly in your code, anyone who sees your GitHub repo can steal them.

'Use a .env file to store secrets and .gitignore to keep them private.'

Step-by-step setup

1. Install dotenv (once per environment)

pip install python-dotenv

2. Create a .env file

In your root folder, create a file named .env with values like this:

DB_USER=your_username
DB_PASSWORD=your_password
DB_HOST=localhost
DB_NAME=your_database_name

3. Load .env values in config.py

Create a file config.py that will read these values:

import os
from dotenv import load_dotenv

load_dotenv()

USER = os.getenv("DB_USER")
PASSWORD = os.getenv("DB_PASSWORD")
HOST = os.getenv("DB_HOST")
DATABASE = os.getenv("DB_NAME")

4. Add .env to .gitignore

Edit your .gitignore file to contain this line:

.env

This keeps your secrets out of your GitHub repo.

5. Document it for other users

In your README.md or instructions, tell users:

'Create a .env file with the same keys, but use your own values.'

Summary Table

  • .env - stores secrets
  • .gitignore - prevents Git from uploading secrets
  • load_dotenv() - loads the .env file
  • os.getenv() - accesses values safely
