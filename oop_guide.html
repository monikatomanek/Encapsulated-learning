1. What is OOP?

OOP means organizing your code using classes and objects that model real things. This keeps your code modular, easier to maintain, and scalable.

2. Classes and Objects

Without OOP:

def drive_car(brand, model):
    print(f"{brand} {model} is driving.")

drive_car("Toyota", "Corolla")

With OOP:

class Car:
    def __init__(self, brand, model):
        self.brand = brand
        self.model = model

    def drive(self):
        print(f"{self.brand} {self.model} is driving.")

car1 = Car("Toyota", "Corolla")
car1.drive()

3. Encapsulation

Encapsulation protects the inner state of your objects. Example: prevent speed from being set to a negative number.

class Car:
    def __init__(self, brand, model, speed=0):
        self.brand = brand
        self.model = model
        self.speed = speed

    @property
    def speed(self):
        return self.__speed

    @speed.setter
    def speed(self, value):
        if value < 0:
            self.__speed = 0
        else:
            self.__speed = value

Now the car can't have a negative speed by mistake.

4. Inheritance

class Car:
    def drive(self):
        print("Driving...")

class ElectricCar(Car):
    def charge(self):
        print("Charging...")

âœ… Reuse logic from the Car class.

5. Polymorphism

class ManualCar(Car):
    def drive(self):
        print("Manual drive")

class AutoCar(Car):
    def drive(self):
        print("Automatic drive")

Even though they all have drive(), the result is different. That's polymorphism.

6. Abstraction

from abc import ABC, abstractmethod

class Vehicle(ABC):
    @abstractmethod
    def drive(self):
        pass

class Car(Vehicle):
    def drive(self):
        print("Car drives")

âœ… Focus on 'what it does' not 'how'.

7. Class and Static Methods

class Car:
    total_cars = 0

    def __init__(self):
        Car.total_cars += 1

    @classmethod
    def show_total_cars(cls):
        print(cls.total_cars)

    @staticmethod
    def convert_speed(kph):
        return kph * 0.621371
